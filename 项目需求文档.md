## 1. 项目概述

### 1.1 项目名称
社区问答系统（Community Q&A System）

### 1.2 项目背景
互联网时代用户产生大量碎片化问题，传统搜索引擎难以提供精准、可信的回答。社区问答系统通过用户互助问答机制，聚合群体智慧，形成可沉淀、可检索的结构化知识库。本系统在传统问答功能之上，引入知识关联推荐、用户成长体系、邀请回答、精华沉淀、敏感词过滤等机制，构建一个兼具实用性与社区活力的知识共享平台。

### 1.3 建设目标
构建一个以问答为核心、融合知识管理与社区运营的社区平台，支持用户提问、回答、评论、点赞、收藏、关注、邀请回答等功能，通过敏感词过滤+审核双重机制保障内容质量，通过等级体系激励用户贡献，通过知识关联与精华沉淀实现内容的结构化复用。

### 1.4 适用用户
- 普通用户：提出问题、浏览内容、参与互动、积累等级
- 管理员：内容审核、用户管理、分类标签管理、敏感词库管理、精华管理、数据统计

### 1.5 系统边界
- **包含：** 用户注册登录、提问、回答（富文本，图片 Base64 内嵌）、评论（二级）、搜索、点赞收藏关注、邀请回答、举报、敏感词自动过滤、内容审核、首页推荐、知识关联推荐、用户等级与成就（预设徽章）、精华区、热榜、数据导出、管理后台
- **不包含：** 第三方登录（微信/QQ/手机号）、邮箱注册/验证、即时通讯（IM）、视频直播、电商交易、支付/积分交易、OSS / 本地文件存储

### 1.6 系统创新点

| 创新点 | 说明 | 解决的问题 |
|--------|------|-----------|
| **敏感词自动过滤** | 用户提交内容时自动扫描敏感词库，命中后阻止提交或标记待审核，支持管理员动态维护词库 | 提升内容安全自动化程度，降低审核人力成本 |
| **知识关联推荐** | 基于标签共现和分类关系，在问题详情页推荐相关问题 | 打破信息孤岛，提升知识发现效率 |
| **用户等级与成就体系** | 贡献度驱动的10级成长体系 + 10种预设徽章 + 排行榜 | 解决激励不足，提升社区活跃度和内容质量 |
| **邀请回答机制** | 提问者指定邀请用户回答，被邀请者收到专属通知 | 解决冷启动困境，加速优质回答产生 |
| **精华知识沉淀** | 管理员标记高质量问答为"精华"，独立检索栏目 | 防止优质内容被时间淹没，形成持久化知识资产 |
| **实时热榜** | 日/周/月热榜，综合多维热度指标定时刷新 | 提升内容曝光效率 |
| **智能标签推荐** | 输入标题后自动推荐匹配标签 | 降低提问门槛，提升标签规范性 |

## 3. 功能模块

### 3.1 用户注册与登录
| 项目 | 内容 |
|------|------|
| **模块目标** | 提供用户身份认证，控制功能访问权限 |
| **使用角色** | 所有用户 |
| **功能点** | ① 用户名+密码注册 ② 登录 ③ 退出登录 ④ 记住登录状态 |
| **输入** | 用户名、密码 |
| **输出** | 登录成功 → 跳转首页；登录失败 → 错误提示 |
| **业务规则** | 用户名唯一、密码最小6位；JWT Token 有效期 7 天；无邮箱、无第三方登录 |

### 3.2 个人中心
| 项目 | 内容 |
|------|------|
| **模块目标** | 用户查看和管理个人信息、历史记录、等级成就 |
| **使用角色** | 普通用户 |
| **功能点** | ① 查看/编辑个人资料（头像 Base64、昵称、简介） ② 我的提问列表 ③ 我的回答列表 ④ 我的收藏 ⑤ 我关注的人/关注我的人 ⑥ 修改密码 ⑦ 等级与成就展示（当前等级、经验值进度条、已解锁徽章墙） |
| **输入** | 昵称、简介、头像（Base64 编码） |
| **输出** | 个人信息展示、等级进度条、成就徽章墙、列表分页展示 |
| **业务规则** | 昵称可重复；头像 Base64 最大 500KB（约 200×200 像素） |

### 3.3 问题管理
| 项目 | 内容 |
|------|------|
| **模块目标** | 用户发布、浏览、搜索问题；支持邀请回答；敏感词自动过滤 |
| **使用角色** | 普通用户、管理员 |
| **功能点** | ① 发布问题 → 敏感词过滤 → 审核 ② 智能标签推荐 ③ 邀请回答 ④ 问题列表（推荐/热门/最新/热榜） ⑤ 搜索 ⑥ 问题详情（含回答列表、相关问题推荐） ⑦ 知识关联推荐 ⑧ 编辑问题（仅作者） ⑨ 删除问题（作者或管理员） |
| **输入** | 问题标题、正文、分类、标签、被邀请用户ID列表 |
| **输出** | 问题列表（分页）、问题详情+关联推荐、搜索匹配结果 |
| **业务规则** | 标题必填、正文可选、分类必选、标签可选（最多5个）；发布时先过敏感词过滤，命中敏感词则阻止提交并提示；通过后 status=pending 进入审核；编辑已发布问题不触发重新审核 |

### 3.4 回答管理
| 项目 | 内容 |
|------|------|
| **模块目标** | 用户对问题提交回答，支持采纳与排序 |
| **使用角色** | 普通用户、管理员 |
| **功能点** | ① 发布回答（富文本，图片 Base64 内嵌）→ 敏感词过滤 → 审核 ② 采纳回答（仅提问者） ③ 编辑回答（仅作者） ④ 删除回答（作者或管理员） ⑤ 回答排序（默认点赞数、时间） |
| **输入** | 回答正文（富文本），图片 Base64 内嵌于正文中 |
| **输出** | 回答列表、采纳状态标识 |
| **业务规则** | 每问题只有一个被采纳的回答，采纳后置顶；富文本支持 Markdown + Base64 内嵌图片；单张图片 Base64 解码后 ≤ 2MB，每回答最多 10 张；富文本 XSS 白名单过滤；敏感词命中 → 阻止提交并提示；被驳回后作者修改 → 自动重置为 pending 重新进入审核 |

### 3.5 评论功能
| 项目 | 内容 |
|------|------|
| **模块目标** | 用户对回答发表评论，支持二级回复；敏感词过滤 |
| **使用角色** | 普通用户 |
| **功能点** | ① 对回答发表评论 → 敏感词过滤 ② 回复评论（二级嵌套） ③ 删除自己的评论 |
| **输入** | 评论正文 |
| **输出** | 评论列表（二级嵌套、按时间排序） |
| **业务规则** | 最大 500 字；不支持三级及以上嵌套；评论范围限定为「回答」；敏感词命中 → 阻止提交并提示 |

### 3.6 互动功能（点赞、收藏、关注）
| 项目 | 内容 |
|------|------|
| **模块目标** | 增强社区互动，挖掘优质内容 |
| **使用角色** | 普通用户 |
| **功能点** | ① 点赞/取消点赞问题 ② 点赞/取消点赞回答 ③ 收藏/取消收藏问题 ④ 关注/取消关注用户 |
| **输入** | 目标ID + 操作类型 |
| **输出** | 点赞数/收藏数/关注数实时更新 |
| **业务规则** | 同一用户对同一目标只能点赞一次；点赞/收藏数公开；关注关系公开 |

### 3.7 邀请回答
| 项目 | 内容 |
|------|------|
| **模块目标** | 提问者主动邀请用户回答，加速问题解决 |
| **使用角色** | 普通用户（提问者） |
| **功能点** | ① 发布问题时搜索并选择被邀请用户（最多3人） ② 被邀请者收到专属通知 ③ 问题详情页展示邀请记录 |
| **输入** | 被邀请用户ID列表 |
| **输出** | 邀请成功 → 通知被邀请者 |
| **业务规则** | 仅提问者可邀请；每问题最多3人；不能邀请自己；被邀请者无强制回答义务 |

### 3.8 举报功能
| 项目 | 内容 |
|------|------|
| **模块目标** | 用户举报违规内容，辅助管理员维护社区秩序 |
| **使用角色** | 普通用户（举报）、管理员（处理） |
| **功能点** | ① 举报问题/回答/评论（选择原因：广告/色情/辱骂/侵权/其他） ② 管理员查看举报列表 ③ 管理员处理（忽略/删除内容/封禁用户） |
| **输入** | 举报目标类型+ID、举报原因、补充说明（可选） |
| **输出** | 举报成功提示；管理员端举报列表 |
| **业务规则** | 同一用户对同一目标只能举报一次；举报不自动删除内容，由管理员人工裁决；处理结果通知举报者 |

### 3.9 分类与标签管理
| 项目 | 内容 |
|------|------|
| **模块目标** | 组织问题内容，便于检索与归类 |
| **使用角色** | 管理员（增删改）、普通用户（选择使用） |
| **功能点** | ① 管理员创建/编辑分类（多级树形） ② 管理员删除分类（级联删除子分类） ③ 管理员创建/编辑/删除标签 ④ 用户发布问题时选择分类和标签 |
| **输入** | 分类名称、父分类ID、标签名称 |
| **输出** | 分类树、标签列表 |
| **业务规则** | 分类支持多级；删除父分类时级联删除所有子分类；分类下有已发布问题时禁止删除；标签仅管理员创建 |

### 3.13 首页推荐
| 项目 | 内容 |
|------|------|
| **模块目标** | 为用户个性化推荐优质内容，提升首页留存 |
| **使用角色** | 所有用户 |
| **功能点** | ① 推荐 feed（综合热度+时间衰减排序） ② 热门 tab ③ 最新 tab ④ 分类导航入口 ⑤ 热榜入口 |
| **输入** | 用户行为加权统计 |
| **输出** | 推荐问题列表（分页） |
| **业务规则** | score = (view_count × 0.3 + answer_count × 1 + vote_count × 2 + favorite_count × 1.5) × time_decay；time_decay：≤24h=1.0，之后每天×0.85，最低0.1；精华问题 score ×1.5 加权；仅推荐 published 状态 |

### 3.14 热榜
| 项目 | 内容 |
|------|------|
| **模块目标** | 展示当前最受关注的问题，引导流量 |
| **使用角色** | 所有用户 |
| **功能点** | ① 日榜（24h） ② 周榜（7天） ③ 月榜（30天） |
| **输入** | 系统定时计算 |
| **输出** | 热榜列表（排名+标题+热度值） |
| **业务规则** | 热度 = view_count × 1 + answer_count × 3 + vote_count × 5 + favorite_count × 3；仅统计 published 问题；每日凌晨刷新 |

| **功能点** | ① 管理员标记/取消精华 ② 精华问题列表独立页面 ③ 精华区搜索 ④ 精华标识展示 |
| **输入** | 管理员操作 |
| **输出** | 精华列表、精华标识 |
| **业务规则** | 仅 published 问题可加精；精华问题在首页推荐中 ×1.5 加权 |

### 3.16 用户等级与成就体系
| 项目 | 内容 |
|------|------|
| **模块目标** | 量化用户贡献，激励持续参与 |
| **使用角色** | 所有登录用户 |
| **功能点** | ① 经验值累计 ② 10级等级制度 ③ 10种预设成就徽章自动解锁 ④ 排行榜（按经验值排名） |
| **输入** | 用户操作事件 |
| **输出** | 等级展示、徽章墙、排行榜 |

**经验值规则：**

| 行为 | 经验值 |
|------|--------|
| 提问（审核通过后） | +1 |
| 回答（审核通过后） | +3 |
| 回答被采纳 | +10 |
| 被点赞（问题/回答） | +2 |
| 被收藏（问题） | +1 |

**等级区间：**

| 等级 | 所需经验 |
|------|----------|
| Lv.1 | 0 ~ 99 |
| Lv.2 | 100 ~ 299 |
| Lv.3 | 300 ~ 599 |
| Lv.4 | 600 ~ 999 |
| Lv.5 | 1,000 ~ 1,999 |
| Lv.6 | 2,000 ~ 3,999 |
| Lv.7 | 4,000 ~ 6,999 |
| Lv.8 | 7,000 ~ 11,999 |
| Lv.9 | 12,000 ~ 19,999 |
| Lv.10 | 20,000+ |

**预设成就徽章（10种，不支持自定义）：**

| # | 徽章名称 | 触发条件 |
|---|----------|----------|
| 1 | 初来乍到 | 完成注册 |
| 2 | 首次提问 | 第一个问题审核通过 |
| 3 | 首次回答 | 第一个回答审核通过 |
| 4 | 最佳答案 | 回答首次被采纳 |
| 5 | 百赞达人 | 累计获得 100 个赞 |
| 6 | 人气之星 | 累计获得 500 个赞 |
| 7 | 知识贡献者 | 累计发布 50 个回答 |
| 8 | 社区常客 | 连续登录 7 天 |
| 9 | 铁杆粉丝 | 累计登录 30 天 |
| 10 | 万人迷 | 累计获得 100 个关注者 |

**业务规则：** 经验值只增不减（删除内容不扣）；成就触发后立即解锁并通知用户；等级和徽章在个人中心+排行榜展示

### 3.17 知识关联推荐
| 项目 | 内容 |
|------|------|
| **模块目标** | 构建问题之间的知识关联，提升知识发现效率 |
| **使用角色** | 所有用户（浏览问题详情时触发） |
| **功能点** | ① 问题详情页侧栏展示"相关问题"（3~5条） ② 关联规则：同分类 + 标签重合度 |
| **输入** | 当前问题分类ID、标签ID列表 |
| **输出** | 关联问题列表（标题+回答数+时间） |
| **业务规则** | 关联度 = (分类相同 × 2) + (标签重合数 × 3)；按关联度降序取 top5；排除自身；仅推荐 published 问题 |

### 3.18 敏感词过滤

## 4. 业务流程

### 4.1 用户提问流程（含敏感词过滤 + 审核）

```
用户登录 → 点击"提问" → 输入标题
→ 系统分析标题 → 推荐匹配标签（用户可点选/忽略）
→ 填写正文 → 选择分类 → 选择标签 → （可选）搜索并邀请用户
→ 提交 → 前端敏感词扫描
  → 命中 → 提示"内容包含不当用语，请修改"，阻止提交
  → 未命中 → 提交到后端 → 后端再次扫描敏感词
    → 命中 → 返回错误提示
    → 未命中 → 保存问题（status=pending）→ 跳转问题详情（仅作者可见）
→ 管理员审核 → 通过 → status=published，公开可见，通知作者
              → 驳回 → status=rejected，通知作者（含驳回原因），作者可修改后重新提交
异常：未登录 → 跳转登录页；标题为空 → 提示必填
```

### 4.2 用户回答流程（含敏感词过滤 + 审核 + Base64 图片）

```
用户浏览问题详情 → 点击"写回答" → 编辑富文本正文（可嵌入 Base64 图片）
→ 提交 → 前端敏感词扫描 → 未命中 → 提交到后端
→ 后端敏感词扫描 → 未命中 → 校验图片（单张 ≤ 2MB Base64，总数 ≤ 10张）
→ 保存回答（status=pending）→ 仅作者可见 → 通知提问者
→ 管理员审核 → 通过 → status=published，公开可见，通知回答者
              → 驳回 → status=rejected，通知回答者（含驳回原因）
              → 回答者修改内容 → 自动重置为 pending → 重新进入审核队列
异常：未登录 → 跳转登录页；正文为空 → 提示必填；敏感词命中 → 提示修改
```

### 4.3 采纳回答流程

```
提问者查看自己的问题详情 → 在回答处点击"采纳"
→ 系统校验：该问题暂无被采纳回答 → 标记 is_accepted=true
→ 回答置顶展示"已采纳"标识 → 通知回答者（含 +10 经验值提示）
异常：非提问者 → 按钮不显示；已有采纳 → 提示"已采纳过"
```

### 4.4 点赞/收藏流程

```
用户浏览问题/回答 → 点击点赞按钮
→ 系统校验是否已点赞
  → 未点赞 → 点赞数+1，按钮高亮，被点赞者经验+2
  → 已点赞 → 点赞数-1，按钮还原，经验不变
收藏逻辑同理（收藏不涉及经验值）
```

### 4.5 内容审核流程

```
用户发布问题/回答 → status=pending → 进入管理员审核队列
→ 管理员在审核列表查看内容详情 →
  → 点击"通过" → status=published → 内容公开可见 → 通知作者
  → 点击"驳回" → 填写驳回原因（必填）→ status=rejected → 通知作者
→ 作者查看被驳回的内容 → 编辑修改 → 保存 → status 自动重置为 pending → 重新进入审核队列
```

**状态流转图：**

```
           ┌──────────┐
           │  pending  │ ← 用户发布 / 被驳回后修改
           └─────┬─────┘
          ┌──────┴──────┐
          ↓              ↓
   ┌──────────┐   ┌──────────┐
   │published │   │ rejected │
   └──────────┘   └─────┬────┘
        ↓               │
   ┌──────────┐         │  作者编辑修改
   │ deleted  │         │  自动重置为 pending
   └──────────┘         │
                   ┌────┴────┐
                   │ pending  │
                   └─────────┘
```

### 4.6 敏感词过滤流程

```
用户提交内容（问题/回答/评论）→ 
  前端：遍历敏感词库（缓存），文本包含任一敏感词 → 阻止提交，提示用户修改
  前端放行 → 后端：查询数据库敏感词表（status=active），文本 contains 匹配
    → 命中 → 返回错误"内容包含不当用语，请修改"
    → 未命中 → 正常进入保存/审核流程

管理员维护词库：
  进入管理后台 → 敏感词管理 → 添加新词（立即生效）/ 禁用某词（立即失效）/ 删除某词
```

### 4.7 举报处理流程

```
用户浏览内容 → 点击"举报" → 选择举报原因 → 提交
→ 系统记录举报（内容状态不变，不自动删除）
→ 管理员在举报列表查看 → 
  → 忽略 → 标记"已处理-忽略" → 通知举报者
  → 删除内容 → 内容软删除 + 标记"已处理-删除" → 通知举报者和作者
  → 封禁用户 → 用户封禁 + 相关内容删除 → 通知举报者
```

### 4.8 用户封禁流程

```
管理员进入用户管理 → 选择用户 → 封禁
→ 系统将用户状态设为 banned → 用户下次请求时 JWT 校验失败/状态校验拒绝操作
→ 管理员可随时解封 → 用户恢复操作权限
```

### 4.9 分类删除流程（级联删除）

```
管理员进入分类管理 → 选择某分类 → 点击删除
→ 系统校验：该分类及其子分类下是否有 published 问题？
  → 有 → 提示"该分类下有已发布问题，无法删除"
  → 无 → 删除该分类 + 级联删除所有子孙分类 → 提示"已删除X个分类"
```

### 4.10 异常流程汇总

| 场景 | 处理方式 |
|------|----------|
| 未登录操作需要登录的功能 | 跳转登录页 |
| JWT 过期 | 返回 401，前端跳转登录页 |
| 非作者编辑/删除他人内容 | 操作按钮不显示 |
| 重复点赞 | 后端去重，返回当前状态 |
| 标题/正文为空提交 | 前端校验+后端兜底 |
| 敏感词命中 | 阻止提交，提示"内容包含不当用语，请修改" |
| 搜索无结果 | 显示"未找到相关问题" |
| 被删除/封禁内容访问 | 返回"内容不存在" |
| 访问待审核/驳回内容 | 仅作者和管理员可查看 |
| Base64 图片超限（单张>2MB/总数>10张） | 前端提示+后端拦截 |
| 邀请自己 | 提示"不能邀请自己" |
| 重复举报同一内容 | 提示"您已举报过该内容" |

### 6.2 数据校验
- 前后端双重校验，前端即时反馈，后端兜底
- **敏感词过滤**：前后端各执行一次，前端缓存词库即时提示，后端数据库查验兜底
- 富文本 XSS 白名单过滤：p / br / b / i / em / strong / a / img / h1~h6 / ul / ol / li / code / pre / blockquote
- SQL 注入防护（参数化查询）
- 分页参数校验（page ≥ 1，pageSize 上限 50）
- 头像 Base64 校验（≤ 500KB）
- 回答图片 Base64 校验（单张解码后 ≤ 2MB，每回答 ≤ 10张）

### 6.3 操作反馈
- 所有增/删/改操作给出明确成功/失败提示
- 网络异常统一错误提示
- 敏感操作（删除问题/回答、封禁用户）弹窗二次确认
- 审核驳回需展示驳回原因
- 举报提交后提示"举报已提交，管理员会尽快处理"
- 敏感词拦截提示"内容包含不当用语，请修改"

### 6.4 查询与分页
- 所有列表类接口分页（page + pageSize，默认 pageSize=20）
- 首页：推荐 feed + 热门 + 最新 三个 tab
- 搜索支持关键词 + 分类 + 标签组合筛选
- 管理后台所有列表支持关键词搜索

### 6.5 性能要求
- 问题列表首页查询 < 200ms
- 搜索响应 < 500ms
- 敏感词过滤（内存缓存 + 数据库兜底）< 10ms
- 统计图表/热榜每日凌晨预计算缓存

### 6.7 后端接口要求
- RESTful 风格，统一响应 `{code, message, data}`
- 登录态：JWT Token，有效期 7 天
- 鉴权中间件：登录校验 + 管理员接口 role 校验
- 文件相关：**无独立文件上传接口**，图片以 Base64 内嵌于富文本正文中
- 敏感词过滤：独立工具类/中间件，所有提交接口（问题/回答/评论）统一调用

### 6.8 数据库要求
- MySQL，字符集 utf8mb4
- 表名小写 + 下划线
- 必备字段：id、created_at、updated_at
- 外键建索引
- question/answer 表 status 字段建索引（审核列表高频查询）
- notification 表 expire_at 建索引（定时清理）
- report 表 status 建索引（举报列表查询）
- sensitive_word 表 status 建索引 + word 字段建唯一索引

### 6.9 定时任务
