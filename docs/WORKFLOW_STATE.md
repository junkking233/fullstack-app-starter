## 更新规则

1. 每次开始开发前，先读取 `docs/AGENTS.md`、`docs/DEVELOPMENT_WORKFLOW.md`、本文件、`docs/GOAL_PLAN.md`（如存在）和当前任务相关文档。
2. 每进入一个新阶段，先把“当前阶段”和“当前任务”更新为本轮要做的工作。
3. 每完成一个阶段，必须更新“最近完成”、“下一步”、“阶段状态表”和必要的证据路径。
4. 如果遇到阻塞，必须写清阻塞项、影响范围、需要用户确认的问题和建议下一步。
5. 不要把本文件当总结文档扩写；它只记录状态、阶段、证据和下一步。
6. 如果用户只要求一个局部模块或修复任务，也要在本文件记录当前任务、涉及阶段和完成状态。
