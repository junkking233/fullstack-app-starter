# 世界杯赛事信息系统需求文档

## 项目概述

项目名称：世界杯赛事信息系统

项目目标：建设一个面向 2026 FIFA 世界杯赛事信息展示与后台维护的 Web 系统。前台面向游客和注册用户，提供球队、赛程、比赛详情、主办城市、场馆、积分榜、淘汰赛对阵图、收藏和比赛评论功能；后台面向管理员，提供用户、球队、赛程、比赛结果、积分榜、淘汰赛结果、评论审核和 ECharts 数据统计能力。

系统边界：
- 不做购票、支付、直播、视频播放。
- 不做球员数据维护。
- 不做新闻资讯模块。
- 不做 Excel 导入。
- 城市和场馆信息来自 FIFA 官方或可靠平台数据，只展示，不提供后台人工维护入口。
- 球队、赛程、比分、积分榜、淘汰赛和评论由管理员后台维护。

数据来源：优先参考 FIFA 官方公开信息，包括赛事日期、48 支球队、12 个小组、104 场比赛、16 个主办城市和晋级规则。平台保留数据来源和更新时间字段，并提供后台数据维护记录，方便后续通过 AI 编程辅助获取数据后写入和维护。

## 用户角色

游客：
- 浏览首页、球队、赛程、比赛详情、主办城市、场馆、积分榜、淘汰赛对阵图和赛事统计。
- 不允许评论和收藏。

注册用户：
- 拥有游客能力。
- 可以收藏比赛和球队。
- 可以在“我的收藏”查看收藏并取消收藏。
- 可以对比赛发表评论。
- 可以查看本人评论审核状态。

管理员：
- 管理用户、球队、赛程、比赛结果、积分榜、淘汰赛结果。
- 审核、驳回、删除比赛评论。
- 查看收藏排行、评论统计、用户统计、赛事统计图表。
- 记录球队、赛程、积分榜、城市场馆等数据的来源、维护动作和备注。
- 不维护城市和场馆。

## 功能模块

登录注册：
- 用户注册、登录、退出、修改密码、查看当前用户。
- 管理员默认账号 `admin/admin123`，普通用户默认账号 `user/123456`。
- 密码使用 MD5，不加 salt。

首页：
- 轮播图、真实赛事概览、近期比赛、热门收藏球队、热门收藏比赛、主办城市入口、积分榜入口、淘汰赛入口。

球队：
- 前台列表、详情、按小组和洲际足联筛选、中英文信息展示、收藏/取消收藏。
- 后台新增、编辑、删除、查询球队。

赛程：
- 前台列表、详情、按日期、球队、小组、阶段、城市筛选。
- 支持比赛收藏和比赛评论。
- 后台新增、编辑、删除、维护比分和状态。

城市与场馆：
- 前台展示城市、场馆和承办比赛。
- 数据仅展示，不提供后台维护入口。

积分榜：
- 按小组展示场次、胜平负、进球、失球、净胜球、积分、排名、晋级状态。
- 小组赛胜 3 分、平 1 分、负 0 分。
- 每组前两名和 8 个最佳小组第三晋级 32 强。系统重算时按积分、净胜球、进球数排序。

淘汰赛对阵图：
- 展示 32 强、16 强、四分之一决赛、半决赛、季军赛、决赛晋级路径。
- 后台维护淘汰赛结果和晋级球队。

比赛评论：
- 注册用户只对比赛发表评论。
- 评论状态：待审核、已通过、已驳回。
- 前台比赛详情只展示已通过评论。
- 用户个人中心展示本人评论审核状态。

收藏：
- 收藏对象类型：比赛、球队。
- 同一用户不能重复收藏同一对象。
- 后台可查看热门球队和热门比赛排行。

图表统计：
- 后台使用 ECharts。
- 包括用户数量、比赛阶段分布、主办城市比赛数量、小组积分、热门收藏球队、热门收藏比赛、评论数量。

数据维护：
- 后台记录数据类型、数据来源、维护动作、操作人、维护备注和时间。
- 数据类型包括球队、赛程、积分榜、城市场馆。
- 维护动作包括手动更新、AI 辅助更新、官方数据核对。

## 非功能需求

- 前端使用 Vue 3 + Vite + npm + Element Plus + ECharts。
- 门户菜单为水平模式，后台菜单为侧栏模式。
- 页面主配色不得使用紫色。
- 后端使用 Spring Boot 3.3.4、JDK 21、MyBatis Plus 3.5.7、MySQL 驱动 8.0.26。
- Java 禁止使用 Lombok。
- 后端统一返回结构：`code`、`message`、`data`、`timestamp`。
- 列表接口支持分页、搜索和筛选。
- 不自动启动前端或后端服务。
- 不重置已有数据库，不创建 MySQL 容器，只连接已有 `mysql-docker`。
