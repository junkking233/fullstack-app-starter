### 4.4 页面设计要求

- 先基于需求文档生成 `docs/原型生图提示词.md`，用户使用 Lovart 生成原型图、交互概览图和 PSD/PNG 后，再进入页面设计拆解。
- 页面设计说明以 Lovart 设计稿为准，输出到 `docs/页面设计.md`。
- 页面设计需覆盖布局、组件、表单字段、列表字段、空状态、加载状态、失败反馈和权限不可见状态。
- Web 前端遵循 Vue + Vite + Element Plus 规范。
- 微信小程序遵循原生微信语法 + Vant Weapp 规范。
- 页面使用图标前必须先检查 `assets/icons/`，缺少图标时先补充共享图标源库，再复制到对应端使用。
- 前端和小程序实现必须对照原型图、PSD、PNG 还原布局、间距、颜色、字号、图标和主要交互状态。
- 禁止使用紫色作为主色。

## 9. 对抗式代码审查清单

- 是否存在需求里要求但接口未覆盖的操作。
- 是否存在接口文档定义但后端未实现的接口。
- 是否存在前端或小程序调用了未定义接口。
- 是否存在跳过 `docs/原型生图提示词.md`、Lovart 设计稿或 `docs/页面设计.md` 直接自由实现页面。
- 是否存在页面布局、颜色、字号、图标、交互状态与 Lovart 原型图、PSD、PNG 明显不一致。
- 是否存在请求参数、响应字段、分页结构或错误码不一致。
- 是否存在数据库表、字段、索引或种子数据无法支撑核心流程。
- 是否存在权限绕过、越权访问或登录态失效处理缺口。
- 是否存在页面只有静态外观但没有真实业务闭环。
- 是否存在底部固定操作栏遮挡、安全区不适配、按钮使用不符合微信小程序规范。
- 是否存在自动启动服务、重置数据库、覆盖用户改动等违规操作。

## 12. 状态同步要求

- 每次执行本计划前，先读取 `docs/WORKFLOW_STATE.md`。
- 每完成一个阶段，更新 `docs/WORKFLOW_STATE.md` 的当前状态、阶段状态表、证据路径和下一步。
