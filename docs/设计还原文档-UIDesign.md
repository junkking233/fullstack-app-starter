只把 PRD ScopeBudget 中 P0/P1 页面列为有效。Figma 中额外存在但不属于 P0/P1 的页面，只记录为“仅参考/暂不开发”。

| 编号 | 页面 | 端 | 关联 P0/P1 功能 | Figma Frame | 匹配来源 | 置信度 | 是否有效 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| P01 |  | Web / 小程序 |  | Figma node 链接 | 用户直给 / metadata 自动匹配 | 高 / 中 / 低 | 有效 / 仅参考 / 缺失 |

匹配规则：

- 用户只提供 Figma 文件链接或 Page 根节点链接时，先读取 Figma metadata，不要直接要求用户逐页复制 Frame 链接。
- 按页面编号、页面中文名、Lovart 文件名和 Figma Frame 名称建立匹配表。
- 优先读取当前 Page 第一层 Frame 树；只有页面名匹配失败时，才少量下钻候选节点。
- 同一页面出现多个候选、找不到候选或命名明显冲突时，才向用户索要具体 Frame 链接。

## 4. 导航与安全区

| 场景 | 规则 |
| --- | --- |
| 一级 Tab 页面 | 必须显示同一组底部 Tab，当前页高亮，内容区预留 Tab 高度 |
| 二级/详情/表单页面 | 默认顶部返回，不显示底部 Tab |
| 固定底部操作区 | 必须加 `env(safe-area-inset-bottom)`，内容区预留底部空间 |
| 微信小程序按钮 | 普通按钮优先用 `view/text + bindtap`，原生 `button` 只用于微信原生能力 |

验收规则：

- `已通过`：代码页面与 Figma Frame 主要视觉一致，只有可接受的小偏差。
- `待修复`：存在明显偏差，必须回到实现阶段修复。
- `阻塞`：缺设计稿、缺资源、无法截图或无法确认，应写清原因。

## 7. 资源规则

- 通用图标先检查根目录 `assets/icons/`。
- Web 图标副本放 `apps/frontend/src/assets/icons/`。
- 小程序图标副本放 `apps/miniprogram/assets/icons/`。
- 缺少图标时先补共享资源库，再复制到子应用。
- 无法直接取出的图片可先用稳定占位并写明阻塞。
