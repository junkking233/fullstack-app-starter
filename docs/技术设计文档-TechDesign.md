## 4. 统一 API 约定

- 响应结构：`code`、`message`、`data`、`timestamp`。
- 成功默认：`code: 200`，`message` 使用中文成功提示，例如 `操作成功`。
- `timestamp` 统一使用 ISO 8601 字符串，例如 `2026-07-10T15:00:00+08:00`；不得与毫秒时间戳混用。
- 默认错误码使用 `400`、`401`、`403`、`409`、`500` 等 HTTP 语义码；如需自定义业务码，必须在本文件统一定义并保持前后端一致。
- 列表接口必须说明分页、筛选和排序。
- 权限接口必须说明登录态、允许角色和越权返回。
- 后端 API 端口固定 `8888`。

## 5. 数据库计划

数据库规则：

- 使用已有 `mysql-docker`，不创建 MySQL 容器。
- 每个业务项目必须使用独立数据库名，导入前检查数据库是否存在和归属；不得使用 `template_db` 作为正式业务库。
- 创建数据库前检查库名是否已存在，不重置已有数据库。
- 字符集使用 UTF-8 / utf8mb4。
- 管理员密码 `admin123`，普通用户密码 `123456`，MD5，不加 salt。
- 数据库变化后立即导出 `db/db.sql`。
- `db/db.sql` 是快照而非迁移系统；已有数据库只执行前向非破坏性变更，禁止导入含 `DROP TABLE` 的 dump。
