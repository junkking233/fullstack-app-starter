## 2. 范围预算 ScopeBudget

范围预算用于防止首版需求过大。PRD 完成前必须先通过本节门禁，再进入 Lovart Prompt。

- 默认角色不超过 3 个；超过时必须合并角色、延期角色或写入待确认问题。
- 每个角色的 P0/P1 核心功能最多 5 个；P2 功能写入暂缓范围，不进入首版原型和实现。
- P0 表示业务闭环必需，P1 表示首版体验必需，P2 表示后续迭代。
- 超出预算时，必须写清保留原因、删减方案和确认状态，不能直接扩大页面、接口或数据库范围。
